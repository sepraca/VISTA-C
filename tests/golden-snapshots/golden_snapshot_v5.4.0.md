## Purpose

Reference baseline captured **before** implementing the direct-surface-illumination / periodic-domain feature (see `TODO-direct-surface-illumination.md`). Generated by importing `physics.js`, `rng.js`, and `simstats.js` directly in Node (no browser needed -- these modules have no DOM dependency for the simulation/accumulation logic itself) and driving the exact same `record()` / `registerCloudBaseTransmission()` / `registerSurfaceEvent()` / `registerSurfaceReflection()` pipeline `runControl.js` uses. Same seed + same physics + deterministic RNG means every number below should reproduce **exactly** (bit-for-bit) after any change that isn't supposed to affect legacy illumination modes. A companion machine-readable `golden_v5.4.0.json` holds full raw counts for automated diffing.

Matrix: 3 illumination modes x 2 solar zenith angles x 3 surface albedos = 18 simulated runs, each recombined into both live Observation geometries post-hoc (a pure aggregation choice -- no re-simulation), for 36 reported rows.

**R6 update (2026-07-14, CODE-REVIEW-v6.0-handoff.md):** this suite originally also recombined each run into a third "Entire scene" geometry (`obsGeom: "scene"`), which existed as dead, UI-unreachable combiner logic (`_bypassInReflected()` in simstats.js) left over from before "Entire scene" was removed from the Observation geometry dropdown pre-v6.0. R6 deleted that dead combiner logic, so "scene" rows are no longer generated by `gen_golden.mjs`/`golden_one.mjs`. This is a test-suite change only: the underlying raw per-photon physics (`rawStats`) and the two surviving combiner rows (`top-base_faces`, `all_faces`) are byte-for-bit identical to the pre-R6 snapshot -- confirmed via `diff_golden.mjs` against a scene-filtered copy of the prior 54-row file before overwriting it. The historical "Entire scene" values are preserved below for the record but are no longer part of the automated regression check.

## How to use this next session

1. After any change to `physics.js` or `simstats.js`, re-run `gen_golden.mjs` unmodified against the updated code.
2. Diff the resulting JSON against `golden_v5.4.0.json` via `tests/review-harness/diff_golden.mjs` for the 18 raw-count blocks / 36 combiner rows.
3. For the three legacy illumination modes (center / top / top_side), every raw count and both live combiner rows must match exactly -- any difference is a real regression, not noise (same seed, deterministic RNG).
4. The new "Uniform domain" illumination and periodic-domain boundary have no legacy baseline to compare against (they're new); validate those against the limiting-case checks in the TODO instead (f_c -> 1 reproduces this same baseline, etc.).
